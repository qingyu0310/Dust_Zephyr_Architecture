# IMU Heater 开环辨识分析

## 1. 实验本质

6 阶段开环阶跃响应辨识：

| 阶段 | 输入 duty | 对应温区 |
|------|-----------|----------|
| run0 | 0.2 | ~37→40°C |
| run1 | 0.3 | ~40→44°C |
| run2 | 0.4 | ~44→48°C |
| run3 | 0.5 | ~48→52°C |
| run4 | 0.6 | ~52→56°C |
| run5 | 0.7 | ~56→62°C |

每个阶段是：**稳态 → 阶跃 Δu → 升温到新的稳态**。

目标是拟合 FOPDT 模型：

```
G(s) = K / (τ·s + 1) · e^{-L·s}
```

其中：

```
K = Δy(∞) / Δu     （最终稳态温升 / 占空比变化）
```

## 2. 为什么不能截取未完成升温的数据直接拟合

假如真实系统：

```
G(s) = 50 / (10s + 1)      Δu = 0.4    ΔT∞ = 20°C
```

起始 30°C，最终 50°C。

截取 30→45°C（未稳态）拟合：

```
K' = 15 / 0.4 = 37.5    误差 25%
```

**K 的估计依赖最终稳态变化量 Δy(∞)。升温没结束就截断，K 一定被低估。**

## 3. 当前 joint fit 为什么能跑出 0.208°C RMSE

因为 6 个阶段都跑到了各自 duty 对应的稳态温度（每段持续 ~15s，τ ≈ 11.7s，> 3τ 已基本稳态）。

所以每段的 ΔT 观测值正确 → K 正确 → RMSE 低。

但是：这是一个 **全局平均模型**，覆盖 37→62°C 整个温区。

## 4. 40°C 模型和 60°C 模型为什么不同

热平衡方程：

```
C · dT/dt = P_in - P_loss
P_loss = h · (T - Ta)    （Ta = 环境温度）
```

| 工作点 | 温差 | 散热 | 相对 40°C 的变化 |
|--------|------|------|------------------|
| 40°C | 15°C | 15h | 基准 |
| 60°C | 35°C | 35h | **散热增加 2.3 倍** |

相同 Δu 下，高温区净加热功率更少 → 升温更慢。

所以在不同温度点做阶跃辨识，得到的是不同的 G(s)：

```
G_40(s) ≠ G_50(s) ≠ G_60(s)
```

当前 joint fit 的 G_mean(s) 是一个折中。

## 5. PID 控温到底需要哪个模型

目标：**40°C 恒温控制**。

控制理论要求：用目标工作点附近的 **局部线性化模型** 设计 PID。

```
G_40(s) = K_40 / (τ_40·s + 1) · e^{-L·s}
```

获得 G_40(s) 的正确实验不是扩大 duty 范围，而是在 40°C 附近做 **小扰动开环阶跃**。

## 6. 正确实验方案

### 方法一：单侧小阶跃（推荐）

1. 手动加热到 40°C 附近并稳定
2. 记录稳定状态的 duty₀ 和温度 T₀
3. 阶跃 Δu（e.g. +0.1）
4. 记录完整升温到新稳态

```
示例：
  u: 0.25 → 0.35
  T: 38°C → 45°C    （跨度 38→45°C，围绕 40°C）
```

工作点 = (0.25 + 0.35)/2 ≈ 0.30，中心温度 ≈ (38 + 45)/2 ≈ 41.5°C。

### 方法二：双向小阶跃（更精确）

在同一工作点做正反两个方向的阶跃，验证线性和滞后对称性：

```
  u₀ - Δu    →    记录降温响应
  u₀ + Δu    →    记录升温响应
```

```
示例：
  u₀ = 0.30
  0.30 → 0.25 → 稳定 → 0.30 → 0.35 → 稳定
```

### 方法三：多组小阶跃联合拟合

与当前 6 轮逻辑相同，但每轮都在 40°C 附近：

```
run1:  0.25 → 0.35    (中心 ~41°C)
run2:  0.28 → 0.38    (中心 ~42°C)
run3:  0.22 → 0.32    (中心 ~40°C)
```

然后用现在的 joint fit 算法拟合 G_40(s)。

## 7. 当前代码怎么用

不需要改任何辨识算法。唯一变化：**实验数据从 6 个宽 duty 阶梯改为 40°C 附近的窄带阶跃**。

具体操作：

1. 在 `Heater::Update()` 的 Normal 模式下，手动干预 duty 做阶跃
2. 或通过 AutoIdent 的 OpenLoop 模式，但修改 `kDutySeq` 为 40°C 附近的值
3. 用现有 `imu_temp_identify.py` 采集和拟合

## 8. 当前全局模型的定位

全局平均模型：

```
G_mean(s) = 42.618 / (11.704·s + 1) · e^{-2.3s}
    ambient = 37.77°C
```

不是错的。它是一个有效模型：

- 用于初步 PID 参数整定（保守设计）
- 用于判断系统的整体时间尺度（τ ≈ 11.7s，L ≈ 2.3s）
- 作为迭代优化的起点

**建议：先拿全局模型跑 PID，跑稳定后再做 40°C 小阶跃实验迭代出更精确的 G_40(s)。**
